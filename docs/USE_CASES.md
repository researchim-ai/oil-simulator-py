# Для чего этот проект: аудитория, сценарии и границы применимости

## 1. Для кого проект

- **Инженерам-разработчикам месторождений** — быстрые сценарии: где бурить, что закачивать, как менять режимы.
- **Студентам и преподавателям** — учебная среда для изучения потоков в пористых средах и численных методов (IMPES, AMG).
- **Исследователям** — платформа для тестирования предобуславливателей, нелинейных решателей (JFNK), новых моделей (EOR).
- **Разработчикам ML/RL** — среда для обучения и оценки алгоритмов управления добычей.

## 2. Типовые сценарии

1. «Пятиточка»: поле 2D/3D, 1 нагнетатель в центре, 4 добывающих по углам. Варьируем дебиты и смотрим фронт вытеснения.
2. «Многоскважинный 3D»: десятки добывающих/нагнетательных, в том числе газовая закачка. Смотрим обводнение и падение дебита.
3. «Мега-кейс» (100×100×100): стресс-тест производительности (AMG + GPU), калибровка параметров и оценка памяти.
4. Сценарный анализ: перестановка скважин, изменение профилей добычи, отключение/включение источников.

## 3. Что «внутри» решается

- Имплицитный шаг давления с AMG-предобуславливателем (GMRES/CG).
- Явное обновление насыщенностей на основе рассчитанных межфазных потоков.
- Пересчет PVT и относительных проницаемостей на каждом шаге.
- Учёт скважин: приток/отток по заданным режимам, перевод дебитов в поверхностные единицы.

## 4. Чем проект НЕ является

- Это не «коробочная» замена промышленным симуляторам: здесь нет всего спектра моделей (corner-point grids, сложные геометрии, много-GPU/DMS по умолчанию).
- Это не готовый продукт для оперативного управления добычей на промысле; но это отличный базис для исследований и обучения.

## 5. Как читать результаты

- Давление и насыщенности — поля по сетке (срезы/карты, 3D-визуализация через VTK).
- Дебиты по скважинам — графики по времени (нефть/вода/газ, обводненность).
- Балансы — проверка суммарной массы и соответствия притоков/оттоков.
- Конвергенция AMG — лог V-циклов, остаток, контролируемые метрики (const_rel, grad_rel).

## 6. Что дальше

- Расширение физики (полимер/термика), JFNK и полностью имплицитная схема.
- Смешанная точность и альтернативный коэрснинг для задач > 1–2 млн ячеек.
- Поддержка много-GPU и распределенного AMG.
